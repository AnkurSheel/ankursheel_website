<html><head></head><body><p>Asteroids was the first game that I could say was more or less feature complete.</p>
<h2 id="what-went-right">What Went Right</h2>
<ul>
<li><strong>Reusing the code:</strong> Since I had gone for a dll-architecture, I did not have to write or even copy-paste most of the basic stuff such as rendering, input, audio etc.</li>
<li><strong>Sticking to the Deadline:</strong> I managed to make a fully playable game with menus and levels within a month.</li>
<li><strong>High Scores:</strong> I finally added a system to show high scores. IF you remember, this was one of the features I had wanted to implement for MPong.</li>
<li><strong>Custom Controls:</strong> Allowing players to customize the controls meant that they were not limited to using WASD.</li>
<li><strong>Menus:</strong> I added a bunch of menus such as HighScores, Controls, Help, Options.</li>
<li><strong>Custom Model Format and ObjFormatter:</strong>, A custom model format meant that I could optimize my code to get the information I needed upfront without running multiple passes. Using the ObjFormatter to pre-process the model file before even starting the game helped improve the loading time.</li>
<li><strong>Collision Detection:</strong> The collision detection algorithm was improved considerably. <strong>Pausing:</strong> Instead of quitting the game directly on the game ending or if the player pressed escape, the options screen would come up, allowing the player to restart or go to the main menu.</li>
</ul>
<h2 id="what-went-wrong">What Went Wrong</h2>
<ul>
<li><strong>Missing Engine Components:</strong> I did not get a chance to work on the physics and networking modules.</li>
<li><strong>Missing Features:</strong> I wanted to implement features such as Shields and alternate weapons, but I ran out of time.</li>
<li><strong>Incomplete Model Loader:</strong> The model loader was not equipped to handle normal's and many other things. This meant that I could not add features like lighting or shadows.</li>
<li><strong>Lack of Data-driven architecture:</strong> The fact that I was not reading the tweakable parameters from a file resulted in long wait times to test the effect of a small value on the overall gameplay.</li>
<li><strong>No Editor for making GUI:</strong> Due to the lack of an editor to make GUI's, I wasted a lot of time customizing the position and scale of elements to make the menus look good. Even after all the work, there was no way to handle multiple resolutions.</li>
<li><strong>No Screen Manager:</strong> I would have loved to handle a screen manager to take care of all the transitions. With the addition of each screen, the amount of maintenance needed to handle the transitions increased exponentially.</li>
</ul>
<h2 id="things-i-wish-i-had-implemented">Things I wish I had Implemented</h2>
<ul>
<li>​Physics and Networking modules.</li>
<li>A better model loader.</li>
<li>Better Graphics.</li>
<li>Pause on Minimizing.</li>
<li>An easier way to edit GUI's.</li>
<li>A credit and splash screen.</li>
</ul>
