<html><head></head><body><p>This post will cover the third-party libraries, project breakup, and current directory structure that I follow. Even though it's not perfect, it has served me well so far.</p>
<h2 id="third-party-tools">Third-Party Tools</h2>
<ul>
<li>libogg and libVorbis- Sound</li>
<li>TinyXML - XML</li>
<li>Zlib - zip</li>
</ul>
<h2 id="projects-in-the-solution">Projects in the Solution</h2>
<ul>
<li><p><strong>Base</strong></p>
<ul>
<li>common data structures and classes</li>
<li>Dependencies - None</li>
<li>Project Includes - None</li>
<li>Additional Includes - None</li>
</ul>
</li>
<li><p><strong>Utilities</strong></p>
<ul>
<li>All Utility Functions</li>
<li>Project Dependencies and Libraries - Base</li>
<li>Project Includes - Base</li>
<li>Additional Includes - TinyXML and Zlib path</li>
</ul>
</li>
<li><p><strong>AI</strong></p>
<ul>
<li>AI-related Functionality</li>
<li>Dependencies - Base, Utilities</li>
<li>Project Includes - Base</li>
<li>Additional Includes - None</li>
</ul>
</li>
<li><p><strong>GraphicsEngine</strong></p>
<ul>
<li>All Graphic related code</li>
<li>Dependencies - Base, Utilities</li>
<li>Project Includes - Base, Utilities</li>
<li>Additional Includes - DirectX Includes</li>
<li>Additional Libraries - None</li>
</ul>
</li>
<li><p><strong>Sound</strong></p>
<ul>
<li>Wrappers for playing sound</li>
<li>Dependencies - Base, Utilities</li>
<li>Includes - Base, Utilities</li>
<li>Additional Includes - DirectX Includes, Libogg path</li>
</ul>
</li>
<li><p><strong>Physics</strong></p>
<ul>
<li>All Physics-related code</li>
<li>Dependencies - Base, Utilities</li>
<li>Includes - Base, Utilities</li>
<li>Additional Includes - None</li>
</ul>
</li>
<li><p><strong>GameBase</strong></p>
<ul>
<li>Functionality that is common and setup for every game.</li>
<li>Dependencies - Base, Utilities, AI, GraphicsEngine, Physics, Sound</li>
<li>Includes - Base, Utilities, AI, GraphicsEngine, Physics, Sound</li>
<li>Additional Includes - None</li>
</ul>
</li>
<li><p><strong>Game</strong></p>
<ul>
<li>GamePlay Logic (<em>We will swap this project for different games</em>)</li>
<li>Dependencies - Base, Utilities, AI, GraphicsEngine, Physics, Sound, GameBase</li>
<li>Includes - Base, Utilities, GraphicsEngine, GameBase, Sound, AI, GameBase</li>
<li>Additional Includes - DirectX includes</li>
</ul>
</li>
<li><p><strong>Main</strong></p>
<ul>
<li>The exe file</li>
<li>Dependencies - Base, Utilities, Game, GameBase</li>
<li>Project Includes - Base, Utilities, GameBase</li>
<li>Additional Includes - None</li>
</ul>
</li>
</ul>
<h2 id="directory-structure">Directory Structure</h2>
<p>For each project, I maintain two folders.</p>
<ul>
<li>Include
<ul>
<li>This folder contains all the interfaces for the respective project. We will include it in all projects that are dependent on it. This way, we will not need to release the underlying implementations. It also reduces the compile-time, if I change the implementation</li>
</ul>
</li>
<li>Src
<ul>
<li>This folder contains all files for the actual implementation. Most of the classes would be inheriting from one of the interfaces in the Include folder</li>
</ul>
</li>
</ul>
<h2 id="final-thoughts">Final Thoughts</h2>
<p>Now that we have the basic directory structure mapped out, the following tutorial will set up the project for the Visual Studio environment.</p>
