<html><head></head><body><p>Sometimes, you want to prevent the compiler from making objects. However, in C++, a copy of an object can be made in many situations. The most obvious is a direct assignment. Less obvious but equally valid is a function call using pass-by-value.</p>
<p>Copying in C++ is handled in two ways.</p>
<ul>
<li>the copy assignment operator; and</li>
<li>the copy constructor</li>
</ul>
<p>Unfortunately, if the class definition does not explicitly declare a copy constructor or a copy assignment operator, the compiler provides an implicit version which is a public member function.</p>
<p>If we want to make a class non-copyable, we can simply define its <strong>copy constructor</strong> and <strong>copy assignment operator</strong> as private members. But, it makes more sense to have a <strong>non-copyable base class</strong>.</p>
<p>If a class derives from the non-copyable base class, it too will be non-copyable. This is because the copy of an object of a derived class must necessarily invoke the copy constructor or the copy assignment operator of its base class.</p>
<pre><code class="language-cpp hljs"><span class="hljs-keyword">namespace</span> Base
{
  <span class="hljs-keyword">class</span> cNonCopyable
  {
  <span class="hljs-keyword">public</span>:
    <span class="hljs-keyword">virtual</span> ~cNonCopyable(){}

  <span class="hljs-keyword">protected</span>:
    cNonCopyable(){}

  <span class="hljs-keyword">private</span>:
    cNonCopyable(<span class="hljs-keyword">const</span> cNonCopyable&amp;);
    cNonCopyable&amp; <span class="hljs-keyword">operator</span> =(<span class="hljs-keyword">const</span> cNonCopyable&amp;);
  };
}
</code></pre>
<p>By making the copy constructor or copy assignment operator private, we ensure that the derived class is non-copyable. We ensure that the compiler does not provide its own versions by providing the declaration and not implementing them. A linking error is generated if used anywhere in the program.</p>
</body></html>
