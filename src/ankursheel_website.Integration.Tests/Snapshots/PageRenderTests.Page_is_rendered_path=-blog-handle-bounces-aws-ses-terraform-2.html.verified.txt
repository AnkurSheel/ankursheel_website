    <h1 class="text-center">How to use terraform to process SQS messages in a lambda</h1>

<html><head></head><body><p>In the last post, we saw <a href="handle-bounces-ses-terraform">how to create an SNS topic to publish the bounce notification to an SQS queue</a>. In this post, we will set up a lambda to process the available messages on the queue.</p>
<h2 id="step-1-create-an-aws-lambda">Step 1: Create an AWS lambda</h2>
<p>We will create the lambda function using <a href="https://registry.terraform.io/providers/hashicorp/aws/latest/docs/resources/lambda_function">Resource: aws_lambda_function</a>.</p>
<pre><code class="language-hcl">resource "aws_lambda_function" "SESBouncesLambda" {
  filename         = "./zips/lambda.zip"
  function_name    = "SESBouncesLambda"
  role             = aws_iam_role.ses_bounces_lambda_role.arn
  handler          = "Lambda::Lambda.Function::FunctionHandler"
  source_code_hash = filebase64sha256("./zips/lambda.zip")
  runtime          = "dotnetcore3.1"
}
</code></pre>
<ul>
<li>Set the file name (including the path) for the deployment package. <em>Create a temporary zip file. We will look at the actual lambda in a later post</em>.</li>
<li>Set the unique name for the lambda.</li>
<li>Set the IAM role for the Lambda function so that we can give it permissions. <em>We will look at this below</em>.</li>
<li>Set the entry-point. <em>This is similar to the main function in a console app</em>.</li>
<li>Set the source code hash so that we can trigger updates for when the function changes.</li>
<li>Set the runtime to <strong>.Net Core 3.1</strong> so that AWS knows that the lambda function is built using .Net core 3.1.</li>
</ul>
<h2 id="step-2-create-an-iam-role-for-the-lambda">Step 2: Create an IAM role for the lambda</h2>
<p>We will use <a href="https://registry.terraform.io/providers/hashicorp/aws/latest/docs/data-sources/iam_policy_document">Data Source: aws_iam_policy_document</a> to allow the lambda service to assume an IAM role to get temporary credentials.</p>
<p>We will use <a href="https://registry.terraform.io/providers/hashicorp/aws/latest/docs/resources/iam_role">Resource: aws_iam_role</a> to set the IAM role for the lambda.</p>
<pre><code class="language-hcl">data "aws_iam_policy_document" "ses_bounces_lambda_role_iam_policy" {
  statement {
    actions = ["sts:AssumeRole"]
    principals {
      type        = "Service"
      identifiers = ["lambda.amazonaws.com"]
    }
  }
}

resource "aws_iam_role" "ses_bounces_lambda_role" {
  name               = "SESBouncesLambdaRole"
  assume_role_policy = data.aws_iam_policy_document.ses_bounces_lambda_role_iam_policy.json
}
</code></pre>
<ul>
<li>Use <em>AssumeRole</em>, which allows AWS services to assume an IAM Role.</li>
<li>Constrain the policy so that only lambdas can assume an IAM role.</li>
<li>Attach the policy to the role.</li>
</ul>
<h2 id="step-3-give-the-lambda-access-to-read-messages-from-sqs-queues">Step 3: Give the lambda access to read messages from SQS queues</h2>
<p>We will use <a href="https://registry.terraform.io/providers/hashicorp/aws/latest/docs/resources/iam_role_policy_attachment">Resource: aws_iam_role_policy_attachment</a> to attach an IAM policy to allow the lambda to read messages from the SQS queue.</p>
<pre><code class="language-hcl">resource "aws_iam_role_policy_attachment" "lambda_sqs_role_policy" {
  role       = aws_iam_role.ses_bounces_lambda_role.name
  policy_arn = "arn:aws:iam::aws:policy/service-role/AWSLambdaSQSQueueExecutionRole"
}
</code></pre>
<ul>
<li>Set the role to which this policy should be attached</li>
<li>Add the AWS managed policy <em>AWSLambdaSQSQueueExecutionRole</em> which has the permissions that the function needs to read items from Amazon SQS and write logs to CloudWatch Logs.</li>
</ul>
<h2 id="step-4-give-the-lambda-access-to-the-queue">Step 4: Give the lambda access to the queue</h2>
<p>We will use <a href="https://registry.terraform.io/providers/hashicorp/aws/latest/docs/resources/lambda_event_source_mapping">Resource: aws_lambda_event_source_mapping</a> to allow our lambda to get events from the SQS queue</p>
<pre><code class="language-hcl">resource "aws_lambda_event_source_mapping" "event_source_mapping" {
  event_source_arn = aws_sqs_queue.ses_bounces_queue.arn
  function_name    = aws_lambda_function.SESBouncesLambda.arn
}
</code></pre>
<ul>
<li>Set the queue to <em>ses_bounces_queue.</em></li>
<li>Set the lambda to <em>SESBouncesLambda</em>.</li>
</ul>
<h2 id="next-steps">Next Steps</h2>
<p>You can see the <a href="https://github.com/AnkurSheel/ses-bounces">code here</a>.</p>
<p>So far, we have configured AWS to publish bounce notifications to an SQS queue. In the next post, we will see how to write a lambda function to send a custom message to a Slack channel.</p>
