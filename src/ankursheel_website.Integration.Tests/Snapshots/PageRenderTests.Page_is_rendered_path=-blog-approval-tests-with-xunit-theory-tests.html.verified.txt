<html><head></head><body><h2 id="problem">Problem</h2>
<p methodname="">In <a href="https://github.com/approvals/ApprovalTests.Net">ApprovalTests</a>, the <strong>Namer</strong> generates a unique name for each approved/received file. This filename takes the format of <strong><em>..approved.txt</em></strong>.</p>
<p>This works perfectly for <strong>Fact</strong> based tests. But, if you are writing a <strong>Theory</strong> based test, the filename will not be unique as a theory-based test is data-driven and therefore has multiple calls to the same method.</p>
<h3 id="example">Example</h3>
<p>Let's look at an example.</p>
<pre><code class="language-csharp hljs"><span class="hljs-keyword">public</span> <span class="hljs-keyword">class</span> <span class="hljs-title">TestClass</span>
{
    [InlineData(<span class="hljs-literal">true</span>)]
    [InlineData(<span class="hljs-literal">false</span>)]
    [Theory]
    <span class="hljs-function"><span class="hljs-keyword">public</span> <span class="hljs-keyword">void</span> <span class="hljs-title">Test</span>(<span class="hljs-params"><span class="hljs-keyword">bool</span> flag</span>)
    </span>{
        <span class="hljs-keyword">var</span> result = TestThisMethod(flag);
        Approvals.Verify(result);
    }
}
</code></pre>
<p>In the above snippet, the filename generated for both the tests will be <strong><em>TestClass.Test.approved.txt</em></strong>. This means that you have only 1 approval file for both tests.</p>
<h2 id="solution">Solution</h2>
<p>The naming pattern of the default namer is <em><strong>[ClassName].[MethodName].[AdditionalInformation(optional)].approved.[Extension]</strong></em>. So, we can leverage the <em>AdditionalInformation</em> part to generate unique filenames for our tests.</p>
<pre><code class="language-csharp hljs"><span class="hljs-keyword">public</span> <span class="hljs-keyword">class</span> <span class="hljs-title">TestClass</span>
{
    [InlineData(<span class="hljs-literal">true</span>)]
    [InlineData(<span class="hljs-literal">false</span>)]
    [Theory]
    <span class="hljs-function"><span class="hljs-keyword">public</span> <span class="hljs-keyword">void</span> <span class="hljs-title">Test</span>(<span class="hljs-params"><span class="hljs-keyword">bool</span> flag</span>)
    </span>{
        NamerFactory.AdditionalInformation = <span class="hljs-string">$"Flag-<span class="hljs-subst">{flag}</span>"</span>;
        <span class="hljs-keyword">var</span> result = TestThisMethod(flag);
        Approvals.Verify(result);
    }
}
</code></pre>
<p>Now, when we run the tests, 2 files will be generated.</p>
<ul>
<li>TestClass.Test.Flag-true.approved.txt</li>
<li>TestClass.Test.Flag-false.approved.txt</li>
</ul>
<p><em>Note: We can pass any information to the <strong>AdditionalInformation</strong> property as long as it is unique between tests.</em></p>
