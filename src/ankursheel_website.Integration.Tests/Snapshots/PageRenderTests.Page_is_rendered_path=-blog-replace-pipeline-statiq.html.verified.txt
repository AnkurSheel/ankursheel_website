<html><head></head><body><p>In this article, I will be showing how we can replace a pipeline in <a href="https://www.statiq.dev/web/">Statiq</a>.</p>
<h2 id="problem">Problem</h2>
<p>As part of the <a href="migrating-gatsby-statiq">migration from Gatsby to Statiq</a>, I wanted to replace the Assets and Content pipeline to control the files, strongly type the model passed to the razor module and also set some metadata based on the parent folder.</p>
<p>Statiq.Web provides quite a few pipelines out of the box. However, it processes different file types by using Templates that define a specific module for that type. Statiq.Web has a nice fluent syntax to modify and remove templates.</p>
<p>In most cases, this would be fine, but sometimes it's much easier to replace the whole pipeline.</p>
<h2 id="solution">Solution</h2>
<p>To remove a pipeline, we can use</p>
<pre><code class="language-csharp hljs">bootstrapper.ConfigureEngine(
    engine =&gt;
    {
        engine.Pipelines.Remove(<span class="hljs-keyword">nameof</span>(Assets));
    });
</code></pre>
<p>To replace it with an empty pipeline, we can use</p>
<pre><code class="language-csharp hljs">bootstrapper.ConfigureEngine(
    engine =&gt;
    {
        engine.Pipelines.Add(<span class="hljs-keyword">nameof</span>(Assets), <span class="hljs-keyword">new</span> Pipeline());
    });
</code></pre>
<p>To replace a pipeline, we can remove it and add it again with a new pipeline.</p>
<pre><code class="language-csharp hljs">bootstrapper.ConfigureEngine(
    engine =&gt;
    {
        engine.Pipelines.Remove(<span class="hljs-keyword">nameof</span>(AnalyzeContent));
        engine.Pipelines.Add(
            <span class="hljs-keyword">nameof</span>(AnalyzeContent),
            <span class="hljs-keyword">new</span> Pipeline
            {
                InputModules =
                {
                    <span class="hljs-keyword">new</span> ReplaceDocuments(
                        <span class="hljs-keyword">nameof</span>(PostPipeline),
                        <span class="hljs-keyword">nameof</span>(Content),
                        <span class="hljs-keyword">nameof</span>(Archives),
                        <span class="hljs-keyword">nameof</span>(Assets))
                }
            });
    });
</code></pre>
<h2 id="conclusion">Conclusion</h2>
<p>Now, we can remove/replace the pipelines that we don't want to use. Dave Glick, the creator of Statiq, has mentioned that he will provide a fluent syntax to replace pipelines in the next release, so hopefully, we won't need this for too long.</p>
<p><em>You can see the original issue I created <a href="https://github.com/statiqdev/Statiq.Web/issues/962">here</a></em></p>
