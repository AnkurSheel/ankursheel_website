<html><head></head><body><p>In this article, I will be showing how we can deploy a website built with <a href="https://tailwindcss.com/">Tailwind CSS</a> and <a href="https://www.statiq.dev/web/">Statiq</a> on <a href="https://www.netlify.com/">Netlify</a>.</p>
<h2 id="problem">Problem</h2>
<p>As part of the <a href="migrating-gatsby-statiq">migration from Gatsby to Statiq</a>, I wanted to deploy my website built with Statiq and Tailwind to Netlify. Netlify allows for only one build command on deployment. The issue is that Tailwind is available as an npm package while Statiq is a .Net solution, and hence both require different commands to run.</p>
<h2 id="solution">Solution</h2>
<p>There are a few ways to do this, but since Statiq Web supports running external processes as part of execution, we will leverage that to install the npm packages and build our statiq website.</p>
<h3 id="step-1-get-the-directories">Step 1: Get the directories</h3>
<pre><code class="language-csharp hljs"><span class="hljs-keyword">var</span> currentDirectory = Directory.GetCurrentDirectory();
<span class="hljs-keyword">var</span> nodeDirectory = Path.Combine(currentDirectory, <span class="hljs-string">".."</span>, <span class="hljs-string">"node"</span>);
<span class="hljs-keyword">var</span> inputDirectory = Path.Combine(currentDirectory, <span class="hljs-string">"input"</span>);
</code></pre>
<ul>
<li><strong>Line 1</strong>: Get the current directory</li>
<li><strong>Line 2</strong>: My npm packages and configs are stored in a node folder, so I need to get the node directory.</li>
<li><strong>Line 3</strong>: Get the input directory where all the razor views are stored.</li>
</ul>
<h3 id="step-2-install-the-npm-packages">Step 2: Install the npm packages</h3>
<pre><code class="language-csharp hljs">bootstrapper.AddProcess(
    ProcessTiming.Initialization,
    _ =&gt; <span class="hljs-keyword">new</span> ProcessLauncher(<span class="hljs-string">"npm"</span>, <span class="hljs-string">"install"</span>)
    {
        LogErrors = <span class="hljs-literal">false</span>,
        WorkingDirectory = nodeDirectory
    })
</code></pre>
<ul>
<li><strong>Line 1</strong>: Adds a new process.</li>
<li><strong>Line 2</strong>: Configures the process to run only once on initialization.</li>
<li><strong>Line 3</strong>: Creates a new process launcher to install the npm packages.</li>
<li><strong>Line 5</strong>: Ignore errors from npm that would fail the build.</li>
<li><strong>Line 6</strong>: Set the working directory to the node directory.</li>
</ul>
<h3 id="step-3-compile-css-with-tailwind">Step 3: Compile CSS with Tailwind</h3>
<pre><code class="language-csharp hljs">bootstrapper.AddProcess(
    ProcessTiming.Initialization,
   _ =&gt; <span class="hljs-keyword">new</span> ProcessLauncher(<span class="hljs-string">"npx"</span>, <span class="hljs-string">"tailwind"</span>, <span class="hljs-string">"build"</span>,
    <span class="hljs-string">$"-i <span class="hljs-subst">{Path.Combine(inputDirectory, <span class="hljs-string">"_site.css"</span>)}</span>"</span>,
    <span class="hljs-string">$"-o <span class="hljs-subst">{Path.Combine(currentDirectory, <span class="hljs-string">"output"</span>, <span class="hljs-string">"styles.css"</span>)}</span>"</span>)
{
    LogErrors = <span class="hljs-literal">false</span>,
    WorkingDirectory = nodeDirectory
});
</code></pre>
<ul>
<li><strong>Line 3</strong>: Use npx to generate a fully compiled Tailwind CSS file.</li>
<li><strong>Line 4</strong>: Set the input path of the custom CSS file to <em>_site.css</em> in the input directory.</li>
<li><strong>Line 5</strong>: Set the path for compiled CSS to the output directory.</li>
</ul>
<h2 id="conclusion">Conclusion</h2>
<p>Now, as part of building the statiq website, the CSS will be generated on each deployment.</p>
