        <article>
            

<div class="prose-sm italic">

<p>Now that we have decided on the basic directory structure and dependencies that each project will have, we will set the project properties. I prefer to do this at the project level so that the properties are persistent, and I don't have to set them every time I download the project. The steps are based on Visual Studio 2010, but you should be able to adapt them to a different IDE.</p>
<h2 id="setting-up-the-main-exe-project">Setting up the Main Exe Project</h2>
<ul>
<li><p>Create a Win 32 Project and delete all the files except for stdafx.h, stdafx.cpp.</p>
</li>
<li><p>Create a new folder called Source and move the above files there (I prefer all my source files in one main directory.</p>
</li>
<li><p>In the property settings for the project, perform the following steps</p>
<ul>
<li>For Both Release and Debug Modes
<ul>
<li>In the tab C/C++ -&gt; General -&gt; Additional Include Directories add</li>
<li>....\Engine\Source\Base\Includes</li>
<li>....\Engine\Source\Utilities\Includes</li>
<li>....\Engine\Source\GameBase\Includes</li>
<li>..\Game\Includes</li>
</ul>
</li>
<li>For Debug Mode
<ul>
<li>In the tab Debugging -&gt; Working Directory add
<ul>
<li>....\Debug\</li>
</ul>
</li>
<li>In the tab Debugging -&gt; Target Name add
<ul>
<li>$(ProjectName)_D</li>
</ul>
</li>
<li>In the tab Linker -&gt; General -&gt; Additional Library Directories add
<ul>
<li>....\Debug\</li>
</ul>
</li>
<li>In the tab Linker -&gt; Input -&gt; Additional Dependencies add
<ul>
<li>Utilities_D.lib</li>
<li>Base_D.lib</li>
<li>Game_D.lib</li>
</ul>
</li>
</ul>
</li>
<li>For Release Mode
<ul>
<li>In the tab Debugging -&gt; Working Directory add
<ul>
<li>....\bin\</li>
</ul>
</li>
<li>In the tab Linker -&gt; General -&gt; Additional Library Directories add
<ul>
<li>....\bin\</li>
</ul>
</li>
<li>In the tab Linker -&gt; Input -&gt; Additional Dependencies add
<ul>
<li>Utilities.lib</li>
<li>Base.lib</li>
<li>Game.lib</li>
</ul>
</li>
</ul>
</li>
</ul>
</li>
</ul>
<h2 id="setting-up-the-dlls">Setting up the DLL's</h2>
<p>I will explain how to set up one of the lib projects, which links to an extern library. Properties for all the other projects can be set in the same way by looking at the dependencies from the <a href="setting-project-directory-structure">previous tutorial</a></p>
<ul>
<li>Create a Win 32 Project but change the Application Type to DLL and delete all the files except for stdafx.h, stdafx.cpp <em>(For tutorial purposes we will be setting up the Utilities project).</em></li>
<li>Create a new folder called Source and move the above files in there.</li>
<li>In the property settings for the project perform the following steps
<ul>
<li>For Both Release and Debug Modes
<ul>
<li>In the tab C/C++ -&gt; General -&gt; Additional Include Directories add
<ul>
<li>Includes
<ul>
<li>..\Base\Includes</li>
</ul>
</li>
</ul>
</li>
<li>In the tab Linker -&gt; General -&gt; Additional Library Directories add
<ul>
<li>....\extern\Lib\Zlib</li>
<li>....\extern\Lib\TinyXml\VS2010</li>
</ul>
</li>
</ul>
</li>
<li>For Debug Mode
<ul>
<li>In the tab Debugging -&gt; Working Directory add
<ul>
<li>....\Debug\</li>
</ul>
</li>
<li>In the tab Debugging -&gt; Target Name add
<ul>
<li>$(ProjectName)_D</li>
</ul>
</li>
<li>In the tab Linker -&gt; General -&gt; Additional Library Directories add
<ul>
<li>......\Debug\</li>
</ul>
</li>
<li>In the tab Linker -&gt; Input -&gt; Additional Dependencies add
<ul>
<li>Base_D.lib</li>
</ul>
</li>
</ul>
</li>
<li>For Release Mode
<ul>
<li>In the tab Debugging -&gt; Working Directory add
<ul>
<li>....\bin\</li>
</ul>
</li>
<li>In the tab Linker -&gt; General -&gt; Additional Library Directories add
<ul>
<li>......\bin\</li>
</ul>
</li>
<li>In the tab Linker -&gt; Input -&gt; Additional Dependencies add
<ul>
<li>Base.lib</li>
</ul>
</li>
</ul>
</li>
</ul>
</li>
</ul>
<h2 id="notes">Notes</h2>
<ul>
<li><p>To point to the DirectX directory, we can use the visual studio macro $(DXSDK_DIR)\Include.</p>
</li>
<li><p>I use the suffix _D to distinguish between debug and release builds.</p>
</li>
<li><p>For the most part, only the Include folder should be added in the additional include directories.</p>
</li>
</ul>
<h2 id="final-thoughts">Final Thoughts</h2>
<p>This finishes the initial setup. Since I am unsure how much detail I should get into why I set up my projects this way, I will be happy to answer any questions in the comments.</p>
