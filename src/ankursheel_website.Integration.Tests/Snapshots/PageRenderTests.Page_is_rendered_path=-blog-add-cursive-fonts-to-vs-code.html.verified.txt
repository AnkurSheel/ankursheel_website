<html><head></head><body><p>I have been using FiraCode as my font of choice in VS Code since I found about it. The use of ligatures while typing code is handy and has been a game-changer. However, lately, I have seen some VS code screenshots with cursive fonts in the text editor, and I wanted some of that. But, I also wanted to keep the ligatures that FiraCode has already spoiled me with.</p>
<p>Using my googling skills, I found a few articles, and it seemed the most straightforward way was to install a theme with a cursive font already built in such as <a href="https://marketplace.visualstudio.com/items?itemName=idbartosz.darkpp-italic">this</a>. However, I like the flexibility of changing themes, so this was not a viable option for me.</p>
<h2 id="final-output">Final Output</h2>
<p><img src="/assets/images/posts/add-cursive-fonts-to-vs-code/cover.png" alt="Final Output Code Snippet"></p>
<p>Now that we know what we are looking to achieve let's see how to make it happen.</p>
<h2 id="updating-vs-codes-settings">Updating VS Code's settings</h2>
<p>My search for a font that encapsulates ligatures and Cursive fonts led me to <a href="https://github.com/kencrocken/FiraCodeiScript">Fira Code iScript</a> made by Ken Krocken. It combines 2 fonts - Fira Code as the regular font and Script12 as the italic font.</p>
<p>After installing the font on my system, I changed the editor settings to use <em>Fira Code iScript</em> as the first option for <em>font family</em>.</p>
<pre><code class="language-json hljs">"editor.fontLigatures": true,
"editor.fontFamily": "'Fira Code iScript', 'Fira Code', Consolas, 'Courier New', monospace",
</code></pre>
<p>VS Code uses the TextMate grammar syntax to define how it needs to render code. You can read more in the <a href="https://code.visualstudio.com/api/language-extensions/syntax-highlight-guide">official documentation</a>. In VS Code <em>settings.json</em>, you can override the font style for the current theme by modifying the <em>textMateRules</em> under <em>editor.tokenColorCustomizations</em>.</p>
<pre><code class="language-json hljs">{
    "editor.tokenColorCustomizations": {
        "textMateRules": [
            {
                "scope": [], // array of Scopes to which the style should be applied to
                "settings": {
                    "fontStyle": "italic" // the font style that should be applied
                }
            }
        ]
    }
}
</code></pre>
<p>You might be wondering how to get the scope to put in the above JSON. Luckily, VS code has a command to help us with this - <strong>Developer: Inspect Editor Tokens and Scopes</strong>. You can trigger this by using the Command Palette (Ctrl + Shift + P) after ensuring that the active cursor is on the token you want to inspect.</p>
<p>The scope inspector displays the following information:</p>
<ul>
<li>The current token.</li>
<li>Metadata about the token and information about its computed appearance.</li>
<li>Theme rules that apply to the token.</li>
<li>Complete scope list, with the most specific scope at the top.</li>
</ul>
<p>The one that we will use is <em>scope list</em>. Whichever token you want to make italic, add it to the scopes in the <em>scopes</em> array.</p>
<p>You can see my current config for an example.</p>
<h2 id="my-current-config">My current config</h2>
<pre><code class="language-json hljs">"editor.tokenColorCustomizations": {
    "textMateRules": [
        {
            "scope": [
            //following will be in italic (=FlottFlott)
                "comment",
                "entity.name.type.class", //class names
                "keyword", //import, export, return…
                "storage.modifier", //static keyword
                "storage.type", //class keyword
                "support.class.builtin",
                "keyword.control",
                "constant.language",
                "entity.other.attribute-name",
                "string.quoted.single",
                "entity.name.method",
                "entity.name.tag"
            ],
            "settings": {
                "fontStyle": "italic",
            }
        }
    ]
}
</code></pre>
<h2 id="conclusion">Conclusion</h2>
<p>After these steps, I have both ligatures and cursive fonts in my editor.</p>
