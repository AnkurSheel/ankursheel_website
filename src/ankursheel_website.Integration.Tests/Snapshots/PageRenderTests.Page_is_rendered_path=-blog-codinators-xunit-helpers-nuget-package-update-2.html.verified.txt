<html><head></head><body><p>I created <a href="https://www.nuget.org/packages/Codinators.XunitHelpers/">Codinators.XunitHelpers</a>(a NuGet package) to add some useful unit testing functionality that is missing in xUnit.</p>
<p>This release adds an auto approver to automatically update the approval files generated by <a href="https://github.com/approvals/approvaltests.net">ApprovalTests.Net</a>.</p>
<h2 id="motivation">Motivation</h2>
<p>We use approval tests for a few scenarios over at FirstAml. Unfortunately, a small typo or a missing dependency registration can result in a lot of failed tests. Not only in this scenario but if we are <a href="approval-tests-with-xunit-theory-tests">using theory tests with Approval files</a> and have it set up to use a <em>DiffReporter</em>, a bunch of windows will open up if the tests fail. We then have to update the approval file and then close the diff window or reject the approval file and close all the windows.</p>
<p>This can be quite annoying and a big time-sink.</p>
<p>The AutoApprover automatically updates the approval files for any failed tests without opening any other windows.</p>
<h2 id="installation">Installation</h2>
<p>Install it by using the following command</p>
<pre><code class="language-bash hljs">dotnet add package Codinators.XunitHelpers
</code></pre>
<h2 id="usage">Usage</h2>
<p>In <code>ApprovalTestConfig.cs</code> add the following snippet</p>
<pre><code class="language-csharp hljs"><span class="hljs-keyword">using</span> xUnitHelpers;

[assembly: UseReporter(<span class="hljs-keyword">typeof</span>(AutoApprover))]
</code></pre>
<h2 id="result">Result</h2>
<p>It will automatically update the approval file. But in the test runner, it will show up as a failing test so that we have a clear indication of which tests failed.</p>
<p>It will not do anything if the received and approved files match so if we run the test again (without changing anything), it will pass.</p>
