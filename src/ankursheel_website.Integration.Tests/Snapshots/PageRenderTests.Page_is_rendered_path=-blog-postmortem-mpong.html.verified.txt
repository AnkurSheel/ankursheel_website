<html><head></head><body><p>The reasons I decided to stop working on Pong before it was completely polished include</p>
<ul>
<li>I had been working on MPong for so long that it was tough to find the motivation to continue working on it.</li>
<li>Although I was continuously adding features to the engine, I was not building up my portfolio.</li>
<li>I wanted to test my engine with another game and see how much the code is reusable.</li>
</ul>
<h2 id="what-went-right">What Went Right</h2>
<ul>
<li><strong>Engine Components</strong>: Starting with Pong as my first game made me concentrate on the engine part. I managed to create a fair amount of the modules I had initially aimed for. Most of the modules were reusable.</li>
<li><strong>Setup</strong>: I managed to develop a relatively painless way of setting up new projects.</li>
<li><strong>Refactoring</strong>: I was pretty happy with how the codebase was at that point (all things considered). The fact that I was continuously refactoring the code was beneficial.</li>
<li><strong>Commenting</strong>: I managed to comment a lot of my classes while programming.</li>
<li><strong>Using a Repository</strong>: Having a repository for my code meant that I could revert any bugs that I introduced without having to waste too much time.</li>
</ul>
<h2 id="what-went-wrong">What Went Wrong</h2>
<ul>
<li><strong>Missing Engine Components</strong>: I did not get a chance to work on the physics and networking modules.</li>
<li><strong>Switching to DirectX 11</strong>: In the middle of the project, I decided to switch to DirectX 11 from DirectX 9. I ended up fixing many bugs that got introduced because of the difference in the way things are handled between the two.</li>
<li><strong>Engine Code as part of Project</strong>: The engine setup and repository was closely tied to the Pong Project. I should have taken out the engine components and made them into a separate repository and solution to be easily reusable.</li>
<li><strong>Not Setting a Deadline</strong>: The fact that I did not have a deadline in place meant that I took my own sweet time to make the game. This has resulted in Mpong being in development for a very long time, without me having to show much for it.</li>
<li><strong>Switching to an Obsolete Technology</strong>: I decided to use DirectSound for my SoundEngine. It was after I had implemented a fair amount of code that someone pointed out to me that I should have used XAudio instead</li>
</ul>
<h2 id="things-i-wish-i-had-implemented">Things I wish I had Implemented</h2>
<ul>
<li>​Physics and Networking modules</li>
<li>Support for 2D game elements.</li>
<li>High Score Table</li>
<li>Better Graphics</li>
<li>Restart Option</li>
<li>Pause on Minimizing</li>
</ul>
