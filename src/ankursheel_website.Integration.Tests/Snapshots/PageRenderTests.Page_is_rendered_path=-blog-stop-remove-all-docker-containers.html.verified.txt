<html><head></head><body><h2 id="problem">Problem</h2>
<p>Occasionally, I have needed to stop and remove all the docker containers I am running locally. I got tired of googling for the commands and wrote a snippet that I could easily access.</p>
<h2 id="solution">Solution</h2>
<p>Before we see how to stop and remove all containers, we need to list all the containers.</p>
<h3 id="list-all-containers">List all Containers</h3>
<p>The command to <a href="https://docs.docker.com/engine/reference/commandline/container_ls/">lists all containers</a> is <code>docker container ls [OPTIONS]</code>.</p>
<p>By default, it shows just the running containers and a lot of information that we won't need. So let's pass in some options.</p>
<ul>
<li><strong><em>-a</em></strong>: Show all the containers.</li>
<li><strong><em>-q</em></strong>: Only display the numeric ids.</li>
</ul>
<pre><code class="language-bash hljs">docker container ls -aq
</code></pre>
<h3 id="stopping-all-containers">Stopping all containers</h3>
<p>The command to <a href="https://docs.docker.com/engine/reference/commandline/container_stop/">stop docker containers</a> is <code>docker container stop [OPTIONS] CONTAINER [CONTAINER...]</code></p>
<p>We can see it takes a space-delimited list of container ids/names to stop. We already have the list of container id's from the previous step. We need to use a shell expansion trick to pass in the list of all containers.</p>
<pre><code class="language-bash hljs">docker container stop $(docker container ls -aq)
</code></pre>
<h3 id="removing-all-containers">Removing all containers</h3>
<p>The command to <a href="https://docs.docker.com/engine/reference/commandline/container_rm/">remove docker containers</a> is <code>docker container rm [OPTIONS] CONTAINER [CONTAINER...]</code></p>
<p>We can see it's similar to the stop command and also takes a space-delimited list of container ids to remove.</p>
<p>We can use the same shell expansion trick to pass in the list of all containers.</p>
<pre><code class="language-bash hljs">docker container rm $(docker container ls -aq)
</code></pre>
<h2 id="conclusion">Conclusion</h2>
<p>This makes it trivial to stop and remove all the containers that I have running locally.</p>
