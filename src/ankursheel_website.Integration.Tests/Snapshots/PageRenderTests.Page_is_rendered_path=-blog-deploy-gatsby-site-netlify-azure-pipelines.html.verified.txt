<p>Deploying a site to Netlify is super simple and has served me well so far. But, it has a few limits, which resulted in my builds failing after my site grew to its current size.</p>
<ul>
<li>a 15-minute timeout limit for builds</li>
<li>the free tier has 300 free minutes in a month. In most cases, this is fine, but gatsby takes a fair amount of time to process images, and I was about to hit the limit.</li>
</ul>
<h2 id="first-attempt-cache-the-gatsby-folders">First Attempt - Cache the gatsby folders</h2>
<p>For some time, I solved the issue by using the unofficial <a href="https://www.npmjs.com/package/gatsby-plugin-netlify-cache">gatsby-plugin-netlify-cache</a> plugin. The plugin uses an <a href="https://www.contentful.com/blog/2018/05/17/faster-static-site-builds-part-one-process-only-what-you-need/#caching-for-the-win">undocumented shared folder</a> to cache the <em>.cache</em> and <em>public</em> folders across builds.</p>
<p>Unfortunately, this solution had 2 main issues.</p>
<ol>
<li>A clear cache and deploy site would fail.</li>
<li>Since my site is image-heavy, I started reaching close to the limit again.</li>
</ol>
<h2 id="second-attempt-use-azure-pipelines">Second Attempt - Use Azure Pipelines</h2>
<p>Azure Pipelines uses a YAML config file to build your project. We will use templates to have separate workflows for <em>pr_and _release</em> builds. Let's create 3 files:</p>
<ol>
<li><em>azure-pipelines.common.yaml</em>: The template file with the job to build the gatsby site and deploy to Netlify.</li>
<li><em>azure-pipelines.pr.yaml</em>: The pipeline to run for PR(Pull Request) builds.</li>
<li><em>azure-pipelines.ci.yaml</em>: The pipeline to run for Live builds.</li>
</ol>
<p>I have separate workflows for Live and PR builds because I do not want PR builds deployed to the live site.</p>
<h3 id="common-job-to-build-gatsby-site">Common Job to Build Gatsby Site</h3>
<p>In <em>azure-pipelines.common.yaml</em>, we create the job to build the gatsby site.</p>
<pre><code class="language-yaml hljs"><span class="hljs-attr">jobs:</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">job:</span> <span class="hljs-string">Build</span>

      <span class="hljs-attr">pool:</span>
          <span class="hljs-attr">vmImage:</span> <span class="hljs-string">'ubuntu-latest'</span>

      <span class="hljs-attr">steps:</span>
          <span class="hljs-bullet">-</span> <span class="hljs-attr">checkout:</span> <span class="hljs-string">self</span>
            <span class="hljs-attr">lfs:</span> <span class="hljs-literal">true</span>

          <span class="hljs-bullet">-</span> <span class="hljs-attr">task:</span> <span class="hljs-string">NodeTool&#64;&#64;0</span>
            <span class="hljs-attr">displayName:</span> <span class="hljs-string">'Install Node.js'</span>
            <span class="hljs-attr">inputs:</span>
                <span class="hljs-attr">versionSpec:</span> <span class="hljs-string">'10.x'</span>

          <span class="hljs-bullet">-</span> <span class="hljs-attr">script:</span> <span class="hljs-string">yarn</span> <span class="hljs-string">install</span>
            <span class="hljs-attr">displayName:</span> <span class="hljs-string">'Install dependencies'</span>

          <span class="hljs-bullet">-</span> <span class="hljs-attr">script:</span> <span class="hljs-string">yarn</span> <span class="hljs-string">build</span>
            <span class="hljs-attr">displayName:</span> <span class="hljs-string">'Build Gatsby'</span>
</code></pre>
<p>We name the job as <em>Build</em> and use the default <em>ubuntu-latest</em> image. This is followed by a series of steps.</p>
<ol>
<li><strong><em>Checkout</em></strong>: Check out the repo and download the files from <em>git-lfs</em>.</li>
<li><strong><em>Install Node.js</em></strong>: Install Node 10.x.</li>
<li><strong><em>Install dependencies</em></strong>: Execute <em>yarn install</em>.</li>
<li><strong><em>Build Gatsby</em></strong>: Execute the <em>yarn build</em> script, which calls <em>gatsby build</em>.</li>
</ol>
<h3 id="live-builds">Live Builds</h3>
<p>Let's add the trigger for live builds in <em>azure-pipelines.ci.yaml</em>.</p>
<pre><code class="language-yaml hljs"><span class="hljs-attr">name:</span> <span class="hljs-string">'Live'</span>

<span class="hljs-attr">trigger:</span>
    <span class="hljs-attr">batch:</span> <span class="hljs-literal">false</span>
    <span class="hljs-attr">branches:</span>
        <span class="hljs-attr">include:</span>
            <span class="hljs-bullet">-</span> <span class="hljs-string">master</span>

<span class="hljs-attr">pr:</span> <span class="hljs-string">none</span>

<span class="hljs-attr">jobs:</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">template:</span> <span class="hljs-string">azure-pipelines.common.yml</span>
</code></pre>
<ol>
<li><strong><em>name</em></strong>: The name for this pipeline.</li>
<li><strong><em>trigger</em></strong>: Conditions for running the pipeline when a push is made.
<ol>
<li><strong><em>batch: false</em></strong>: trigger the pipeline for each commit, i.e. disable batching builds.</li>
<li><strong><em>branches</em></strong>: Include only the <em>master</em> branch, so the pipeline is triggered only for commits pushed to <em>master</em>.</li>
</ol>
</li>
<li><strong><em>pr: none</em></strong>: Disable triggering the pipeline when a PR is opened or changes are pushed to an open PR.</li>
<li><strong><em>jobs</em></strong>: The jobs to run.
<ol>
<li><strong><em>template</em></strong>: Use the job template created in <em>azure-pipelines.common.yml</em>.</li>
</ol>
</li>
</ol>
<h3 id="prpull-requests-builds">PR(Pull Requests) Builds</h3>
<p>I also want a build to be triggered every time a PR is made against <em>master</em>.</p>
<pre><code class="language-yaml hljs"><span class="hljs-attr">name:</span> <span class="hljs-string">'PR $(System.PullRequest.PullRequestId)'</span>

<span class="hljs-attr">trigger:</span>
    <span class="hljs-attr">branches:</span>
        <span class="hljs-attr">exclude:</span>
            <span class="hljs-bullet">-</span> <span class="hljs-string">'*'</span>
<span class="hljs-attr">pr:</span>
    <span class="hljs-attr">autoCancel:</span> <span class="hljs-literal">false</span>
    <span class="hljs-attr">branches:</span>
        <span class="hljs-attr">include:</span>
            <span class="hljs-bullet">-</span> <span class="hljs-string">master</span>

    <span class="hljs-bullet">-</span> <span class="hljs-attr">template:</span> <span class="hljs-string">azure-pipelines.common.yml</span>
</code></pre>
<ol>
<li><strong><em>name</em></strong>: The name for this pipeline.</li>
<li><strong><em>trigger</em></strong>: Conditions for running the pipeline when a push is made.
<ol>
<li><strong><em>branches</em></strong>: Exclude all branches so that the build is not triggered for commits on any branch, including <em>master</em>.</li>
</ol>
</li>
<li><strong><em>pr:</em></strong>: Conditions for running the pipeline when a PR is opened or changes pushed to an open PR.
<ol>
<li><strong><em>autoCancel</em></strong>: Additional pushes should not cancel in-progress runs.</li>
<li><strong><em>include</em></strong>: Include only the <em>master</em> branch so that the pipeline is triggered only for PR's opened against <em>master</em>.</li>
</ol>
</li>
<li><strong><em>jobs</em></strong>: The jobs to run.
<ol>
<li><strong><em>template</em></strong>: Use the job template created in <em>azure-pipelines.common.yml</em>.</li>
</ol>
</li>
</ol>
<p>At this point, you might be wondering about having 2 separate files for pr and Live builds. This will become important in the next section when we add the steps to deploy to Netlify.</p>
<h3 id="deploying-to-netlify">Deploying to Netlify</h3>
<p>To deploy to Netlify, we need a few variables passed to the template as parameters and then passed as flags to the Netlify deploy command.</p>
<p>First, let's see what changes we need to make to the template <em>azure-pipelines.common.yml</em>. The updated lines are highlighted.</p>
<pre><code class="language-yaml hljs"><span class="hljs-attr">parameters:</span>
    <span class="hljs-attr">deployScriptParams:</span> <span class="hljs-string">''</span> <span class="hljs-comment"># defaults for any parameters that aren't specified</span>
    <span class="hljs-attr">netlifySiteId:</span> <span class="hljs-string">'defaultID'</span>
    <span class="hljs-attr">netlifyToken:</span> <span class="hljs-string">'defaultToken'</span>
    <span class="hljs-attr">message:</span> <span class="hljs-string">''</span>

<span class="hljs-attr">jobs:</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">job:</span> <span class="hljs-string">Build_And_Deploy</span>

      <span class="hljs-attr">pool:</span>
          <span class="hljs-attr">vmImage:</span> <span class="hljs-string">'ubuntu-latest'</span>

      <span class="hljs-attr">steps:</span>
          <span class="hljs-comment">#unchanged from before</span>

          <span class="hljs-bullet">-</span> <span class="hljs-attr">script:</span> <span class="hljs-string">yarn</span> <span class="hljs-string">build</span>
            <span class="hljs-attr">displayName:</span> <span class="hljs-string">'Build Gatsby'</span>

          <span class="hljs-comment"># prettier-ignore</span>
          <span class="hljs-bullet">-</span> <span class="hljs-attr">script:</span> <span class="hljs-string">yarn</span> <span class="hljs-string">netlify</span> <span class="hljs-string">deploy</span> <span class="hljs-string">--site</span> <span class="hljs-string">${{</span> <span class="hljs-string">parameters.netlifySiteId</span> <span class="hljs-string">}}</span> <span class="hljs-string">--auth</span> <span class="hljs-string">${{</span> <span class="hljs-string">parameters.netlifyToken</span> <span class="hljs-string">}}</span> <span class="hljs-string">--message</span> <span class="hljs-string">'$<span class="hljs-template-variable">{{ parameters.message }}</span>'</span> <span class="hljs-string">${{</span> <span class="hljs-string">parameters.deployScriptParams</span> <span class="hljs-string">}}</span> <span class="hljs-string">--dir=public</span>
            <span class="hljs-attr">displayName:</span> <span class="hljs-string">'Deploy to Netlify'</span>
</code></pre>
<ol>
<li><strong><em>parameters</em></strong>: The environment variables to pass as flags to the Netlify deploy command.
<ol>
<li><strong><em>deployScriptParams</em></strong>: Any additional parameters that should be passed to the deploy command.</li>
<li><strong><em>netlifySiteId</em></strong>: The site ID for the Netlify site.</li>
<li><strong><em>netlifyToken</em></strong>: The access token to authorize Azure Pipeline to deploy the build to Netlify.</li>
<li><strong><em>message</em></strong>: The message to include in the deploy log.</li>
</ol>
</li>
<li><strong><em>Deploy to Netlify</em></strong>: The Netlify deploy command takes the siteId, token, message and deployScriptParams as flags. We also set the dir flag to public so that the public folder is deployed.
<ol>
<li><strong><em># prettier-ignore</em></strong>: I have added this so that prettier does not try to break up the command into multiple lines.</li>
</ol>
</li>
</ol>
<p>Now we update <em>azure-piplelines.ci.yaml</em> for the live build.</p>
<pre><code class="language-yaml hljs"><span class="hljs-comment"># unchanged from before</span>
<span class="hljs-attr">variables:</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">template:</span> <span class="hljs-string">azure-pipelines.vars.yml</span>

<span class="hljs-attr">jobs:</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">template:</span> <span class="hljs-string">azure-pipelines.common.yml</span>
      <span class="hljs-attr">parameters:</span>
          <span class="hljs-attr">deployScriptParams:</span> <span class="hljs-string">'--prod'</span>
          <span class="hljs-attr">netlifySiteId:</span> <span class="hljs-string">$(netlify.siteId)</span>
          <span class="hljs-attr">netlifyToken:</span> <span class="hljs-string">$(netlify.token)</span>
          <span class="hljs-attr">message:</span> <span class="hljs-string">'Live $(Build.SourceVersion) $(Build.SourceVersionMessage)'</span>
</code></pre>
<ol>
<li><strong><em>variables</em></strong>: We set a template file to get the environment variables.</li>
<li><strong><em>parameters</em></strong>: The parameters to be passed to the job template
<ol>
<li><strong><em>deployScriptParams</em></strong>: I pass the --prod flag for live builds, which tells Netlify to deploy the build to production.</li>
<li><strong><em>netlifySiteId</em></strong>: The site ID from environment variables.</li>
<li><strong><em>netlifyToken</em></strong>: The token from environment variables.</li>
<li><strong><em>message</em></strong>: The message is created using the commit ID and the comment of the commit.</li>
</ol>
</li>
</ol>
<p>Next <em>azure-piplelines.pr.yaml</em> for the PR builds.</p>
<pre><code class="language-yaml hljs"><span class="hljs-comment"># unchanged from before</span>
<span class="hljs-attr">variables:</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">template:</span> <span class="hljs-string">azure-pipelines.vars.yml</span>

<span class="hljs-attr">jobs:</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">template:</span> <span class="hljs-string">azure-pipelines.common.yml</span>
      <span class="hljs-attr">parameters:</span>
          <span class="hljs-attr">netlifySiteId:</span> <span class="hljs-string">$(netlify.siteId)</span>
          <span class="hljs-attr">netlifyToken:</span> <span class="hljs-string">$(netlify.token)</span>
          <span class="hljs-attr">message:</span> <span class="hljs-string">'PR $(System.PullRequest.PullRequestNumber) $(System.PullRequest.SourceBranch) $(Build.SourceVersionMessage)'</span>
</code></pre>
<ol>
<li><strong><em>variables</em></strong>: We set a template file to get the environment variables.</li>
<li><strong><em>parameters</em></strong>: The parameters to be passed to the job template
<ol>
<li><strong><em>netlifySiteId</em></strong>: The site ID from environment variables.</li>
<li><strong><em>netlifyToken</em></strong>: The token from environment variables.</li>
<li><strong><em>message</em></strong>: The message is created using the Pull Request Number, Source branch and the comment of the commit.</li>
</ol>
</li>
</ol>

<p>By using Azure Pipelines, the Netlify limits are no longer a problem. We will see how we can add tests and run them against the deployed URL in a later post.</p>
