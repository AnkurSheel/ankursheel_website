    <h1 class="text-center">Add Tailwind CSS to a Statiq website</h1>
    <div class="flex flex-col min-h-screen">
        <article class="prose-code:bg-slate-400">
            

<div class="prose-sm italic">
    <div class="flex flex-col sm:flex-row justify-evenly items-center text-sm">

        <p class="mr-4">Last Updated on <time datetime="2021-09-09">Sep 09, 2021</time></p>

    <div class="flex flex-row flex-wrap gap-1 place-content-center">

<p>In this article, I will be showing how we can integrate <a href="https://tailwindcss.com/">Tailwind CSS</a> with <a href="https://www.statiq.dev/web/">Statiq</a>.</p>
<h2 id="problem">Problem</h2>
<p>As part of the <a href="migrating-gatsby-statiq">migration from Gatsby to Statiq</a>, I wanted to use Tailwind with Statiq. Tailwind is available as an NPM package, while Statiq is a .Net solution.</p>
<p>Many articles out there discuss the pros and cons of using Tailwind so that I won't be doing that. All I will be saying is that Tailwind is a utility first framework with what might be considered an ugly-as syntax, but boy is it faster to build elegant components.</p>
<h2 id="prerequisites">Prerequisites</h2>
<ul>
<li>Install <a href="https://nodejs.org/en/">Node and npm</a></li>
<li>Install <a href="https://dotnet.microsoft.com/">.Net Core</a></li>
<li>Create a <a href="https://www.statiq.dev/web/">.Net Core Console Application configured to use Statiq</a></li>
</ul>
<h2 id="install-tailwind">Install Tailwind</h2>
<p>Tailwind can only be installed as an NPM package.</p>
<h3 id="step-1-create-a-folder-called-node">Step 1: Create a folder called node</h3>
<p>Because our core project is a .Net Core application, and we don't need npm except for Tailwind, we will install it in a separate folder. This will avoid clutter in the root directory.</p>
<h3 id="step-2-install-packages">Step 2: Install packages</h3>
<p>We will install Tailwind, and other peer dependencies inside a directory called <em>node</em>.</p>
<pre><code class="language-bash hljs"><span class="hljs-built_in">cd</span> node
npm install -D tailwindcss&<span class="hljs-comment">#64;latest postcss&#64;latest autoprefixer&#64;latest</span>
</code></pre>
<ul>
<li><strong>PostCSS</strong> is a preprocessor that transforms the CSS using plugins.</li>
<li><strong>AuoPrefixer</strong> is a PostCSS plugin to add vendor prefixes to CSS rules.</li>
</ul>
<p>Our package.json should look something like this.</p>
<pre><code class="language-json hljs"><span class="hljs-punctuation">{</span>
    &quot;devDependencies&quot;<span class="hljs-punctuation">:</span> <span class="hljs-punctuation">{</span>
        &quot;&#<span class="hljs-number">64</span>;&#<span class="hljs-number">64</span>;tailwindcss/typography&quot;<span class="hljs-punctuation">:</span> &quot;^<span class="hljs-number">0.4</span><span class="hljs-number">.1</span>&quot;<span class="hljs-punctuation">,</span>
        &quot;autoprefixer&quot;<span class="hljs-punctuation">:</span> &quot;^<span class="hljs-number">10.3</span><span class="hljs-number">.1</span>&quot;<span class="hljs-punctuation">,</span>
        &quot;postcss&quot;<span class="hljs-punctuation">:</span> &quot;^<span class="hljs-number">8.3</span><span class="hljs-number">.6</span>&quot;<span class="hljs-punctuation">,</span>
        &quot;tailwindcss&quot;<span class="hljs-punctuation">:</span> &quot;^<span class="hljs-number">2.2</span><span class="hljs-number">.7</span>&quot;
    <span class="hljs-punctuation">}</span>
<span class="hljs-punctuation">}</span>
</code></pre>
<h3 id="step-3-configure-postcss">Step 3: Configure postcss</h3>
<p>Create a postcss.config.js file and add <em>tailwindcss</em> and <em>autoprefixer</em></p>
<pre><code class="language-javascript hljs"><span class="hljs-variable language_">module</span>.<span class="hljs-property">exports</span> = {
    <span class="hljs-attr">plugins</span>: {
        <span class="hljs-attr">tailwindcss</span>: {},
        <span class="hljs-attr">autoprefixer</span>: {},
    },
};
</code></pre>
<h3 id="step-3-configure-tailwind">Step 3: Configure Tailwind</h3>
<p>Generate a config file using the Tailwind CLI utility</p>
<pre><code class="language-bash hljs">npx tailwindcss init
</code></pre>
<p>This will create a tailwind.config.js file.</p>
<p>We will update the purge section to optimize our CSS</p>
<pre><code class="language-js hljs">    <span class="hljs-attr">purge</span>: {
        <span class="hljs-attr">enabled</span>: <span class="hljs-literal">true</span>,
        <span class="hljs-attr">content</span>: [<span class="hljs-string">'../**/input/**/*.cshtml'</span>],
    },
</code></pre>
<ul>
<li><strong>Line 2</strong>: Enable purge without having to set the NODE_ENV to production</li>
<li><strong>Line 3</strong>: Scan our razor views files and remove any superfluous CSS from our final output file. We prefix the path with <strong>../</strong> because our config is inside a subfolder, and we need to find the razor views from the root.</li>
</ul>
<p>Our final configuration will look like this.</p>
<pre><code class="language-js hljs"><span class="hljs-variable language_">module</span>.<span class="hljs-property">exports</span> = {
    <span class="hljs-attr">purge</span>: {
        <span class="hljs-attr">enabled</span>: <span class="hljs-literal">true</span>,
        <span class="hljs-attr">content</span>: [<span class="hljs-string">'../**/input/**/*.cshtml'</span>],
    },
