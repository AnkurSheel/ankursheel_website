    <h1 class="text-center">How do I implement Graphql subscriptions in HotChocolate with Redis</h1>
    <div class="flex flex-col min-h-screen">
        <article>
            
<div class="prose-sm italic ">
    <div class="flex justify-evenly items-center">

            Reading Time: 2 min

<html><head></head><body><p>GraphQL offers us real-time capabilities to notify us when a specific event happens.</p>
<p>In this tutorial, we will add subscriptions to our graphql server. We will create an <code>onPing</code> event that will send notifications whenever we ping the server with an input string.</p>
<p>We will also use Redis that runs in a docker container, and see how we can listen to the events in a Redis command shell.</p>
<h2 id="running-redis-using-docker">Running Redis using docker</h2>
<pre><code class="language-bash hljs">docker run <span class="hljs-_">-d</span> --name=redis-graphql -p 6379:6379 redis:6.0-alpine
</code></pre>
<h2 id="adding-a-subscription">Adding a Subscription</h2>
<p>We will add a new class Subscription which will have our <code>onPing</code> event.</p>
<pre><code class="language-csharp hljs"><span class="hljs-keyword">public</span> <span class="hljs-keyword">class</span> <span class="hljs-title">Subscription</span> : <span class="hljs-title">ObjectType</span>
{
    <span class="hljs-function"><span class="hljs-keyword">protected</span> <span class="hljs-keyword">override</span> <span class="hljs-keyword">void</span> <span class="hljs-title">Configure</span>(<span class="hljs-params">IObjectTypeDescriptor descriptor</span>)
    </span>{
        descriptor.Field(<span class="hljs-string">"onPing"</span>)
            .Type&lt;NonNullType&lt;StringType&gt;&gt;()
            .Resolve(ctx =&gt; ctx.GetEventMessage&lt;<span class="hljs-keyword">string</span>&gt;())
            .Subscribe(<span class="hljs-keyword">async</span> ctx =&gt; <span class="hljs-keyword">await</span> ctx.Service&lt;ITopicEventReceiver&gt;().SubscribeAsync&lt;<span class="hljs-keyword">string</span>, <span class="hljs-keyword">string</span>&gt;(<span class="hljs-string">"onPing"</span>, ctx.RequestAborted));
    }
}
</code></pre>
<ul>
<li><strong><em>Line 5</em></strong>: The topic for the subscription.</li>
<li><strong><em>Line 7</em></strong>: Gets the message payload.</li>
<li><strong><em>Line 8</em></strong>: Subscribes to the pub/sub system</li>
</ul>
<p>To use subscriptions, we need to add the WebSockets middleware to our request pipeline.</p>
<pre><code class="language-csharp hljs"><span class="hljs-comment">// Startup.cs</span>
<span class="hljs-function"><span class="hljs-keyword">public</span> <span class="hljs-keyword">void</span> <span class="hljs-title">Configure</span>(<span class="hljs-params">IApplicationBuilder app, IWebHostEnvironment env</span>)
</span>{
    app.UseWebSockets();
}
</code></pre>
<p><em>Note: Middleware order is important with .NET Core, so this middleware needs to come before the GraphQL middleware.</em></p>
<p>Next, We need to register our subscriptions and set up the Redis subscription provider in the Startup class.</p>
<pre><code class="language-csharp hljs"><span class="hljs-function"><span class="hljs-keyword">public</span> <span class="hljs-keyword">void</span> <span class="hljs-title">ConfigureServices</span>(<span class="hljs-params">IServiceCollection services</span>)
</span>{
    <span class="hljs-keyword">var</span> options = <span class="hljs-keyword">new</span> ConfigurationOptions
    {
        EndPoints = { <span class="hljs-string">"localhost:6379"</span> },
    };

    services.AddGraphQLServer()
            .AddSubscriptionType&lt;Subscription&gt;();
            .AddRedisSubscriptions(_ =&gt; ConnectionMultiplexer.Connect(options));
}
</code></pre>
<p><em>Note: Redis is on port 6379 by default.</em></p>
<h2 id="publishing-the-event">Publishing the event</h2>
<p>Our mutation needs to use <code>ITopicEventSender</code> to send messages to the underlying pub/sub system.</p>
<pre><code class="language-csharp hljs"><span class="hljs-keyword">public</span> <span class="hljs-keyword">class</span> <span class="hljs-title">Mutation</span> : <span class="hljs-title">ObjectType</span>
{
    <span class="hljs-function"><span class="hljs-keyword">protected</span> <span class="hljs-keyword">override</span> <span class="hljs-keyword">void</span> <span class="hljs-title">Configure</span>(<span class="hljs-params">IObjectTypeDescriptor descriptor</span>)
    </span>{
        descriptor.Field(<span class="hljs-string">"ping"</span>)
            .Type&lt;NonNullType&lt;StringType&gt;&gt;()
            .Argument(<span class="hljs-string">"payload"</span>, des =&gt; des.Type&lt;NonNullType&lt;StringType&gt;&gt;())
            .Resolve(
                <span class="hljs-keyword">async</span> ctx =&gt;
                {
                    <span class="hljs-keyword">var</span> input = ctx.ArgumentValue&lt;<span class="hljs-keyword">string</span>&gt;(<span class="hljs-string">"payload"</span>);

                    <span class="hljs-keyword">await</span> ctx.Service&lt;ITopicEventSender&gt;().SendAsync(<span class="hljs-string">"onPing"</span>, input);
                    <span class="hljs-keyword">return</span> input;
                });
    }
}
</code></pre>
<ul>
<li><strong><em>Line 13:</em></strong>: "onPing" is the topic we want to publish, and <em>input</em> is our payload.</li>
</ul>
<h2 id="testing">Testing</h2>
<p>We can start an Interactive Shell for the Redis Docker Container.</p>
<pre><code class="language-bash hljs">docker <span class="hljs-built_in">exec</span> -it redis-graphql redis-cli
</code></pre>
<p>Next, we can subscribe to the <em>onPing</em> channel using the subscribe command.</p>
<pre><code class="language-bash hljs">subscribe onPing
</code></pre>
<p><img src="/assets/images/posts/implement-graqhql-subscriptions-hotchocolate-redis/redis_docker_interactive_shell.png" alt="Subscribing to the topic in an interactive shell for redis"></p>
<p>In Insomnia (or your IDE of choice), add the following mutation.</p>
<pre><code class="language-graphql">mutation {
    ping(payload: "hello")
}
</code></pre>
<p>On execution, we should see the following responses.</p>
<p><img src="/assets/images/posts/implement-graqhql-subscriptions-hotchocolate-redis/mutation.png" alt="Mutation and response"></p>
<p>In the shell, we can see the event on Redis.</p>
<p><img src="/assets/images/posts/implement-graqhql-subscriptions-hotchocolate-redis/redis_response.png" alt="Event caught by Redis listener"></p>
