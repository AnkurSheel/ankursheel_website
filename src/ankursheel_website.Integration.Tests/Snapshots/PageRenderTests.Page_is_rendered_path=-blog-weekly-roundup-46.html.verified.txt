<!-- markdownlint-disable no-duplicate-header --><html><head></head><body><p>Every week I come across some great articles. Here are some of the posts I feel you will enjoy reading as much as I did.</p>
<h2 id="the-pragmatic-engineer-test-12-questions-on-engineering-culture"><a href="https://blog.pragmaticengineer.com/pragmatic-engineer-test/">The Pragmatic Engineer Test: 12 Questions on Engineering Culture</a></h2>
<p><em>by <strong>Gergely Orosz</strong></em></p>
<p>You have probably heard of the <a href="https://www.joelonsoftware.com/2000/08/09/the-joel-test-12-steps-to-better-code/">Joel Test</a>, but it's 20 years old, and most companies can answer <strong>yes</strong> to those questions. These questions are better suited to rate the quality of a software team in today's world.</p>
<h2 id="implementing-content-security-policy-csp-in-asp.net-core"><a href="https://dotnetthoughts.net/implementing-content-security-policy-in-aspnetcore/">Implementing Content Security Policy (CSP) in ASP.NET Core</a></h2>
<p><em>by <strong>Anuraj</strong></em></p>
<p>CSP (Content Security Policy) is a security layer that helps us mitigate and report XSS(Cross-Site Scripting) attacks. This is a guide for creating middleware to set the header to the HTTP response in .Net Core.</p>
<h2 id="how-potty-training-shaped-my-tech-teaching-style"><a href="https://dev.to/shaundai/how-potty-training-shaped-my-tech-teaching-style-29e2">How Potty Training Shaped My Tech Teaching Style</a></h2>
<p><em>by <strong>Shaundai Person</strong></em></p>
<p>Did you know there are parallels between teaching coding and potty training? In both cases, we are giving someone a tool they have never seen before combined with a language they have never heard before and then expecting them to do something they have never done before. This article outlines what can we do differently to avoid these mistakes.</p>
<h2 id="debugging-configuration-values-in-asp.net-core"><a href="https://andrewlock.net/debugging-configuration-values-in-aspnetcore/">Debugging configuration values in ASP.NET Core</a></h2>
<p><em>by <strong>Andrew Lock</strong></em></p>
<p>When working with .Net Core, it can be useful to inspect the resolved configuration settings to determine why a configuration value does not have the value we expect. This post shows how to use <code>GetDebugView()</code> to determine where the configuration values have come from. It also goes through the implementation.</p>
<h2 id="the-three-sides-of-risk"><a href="https://www.collaborativefund.com/blog/the-three-sides-of-risk/">The Three Sides of Risk</a></h2>
<p><em>by <strong>Morgan Housel</strong></em></p>
<p>What can skiing teach you about investing? A lot, it seems. There are distinct sides of risk 2 of which are easy to grasp. But the last one, the tail-end consequences of getting hit, can often be learned only through experience. But the tail-end events are all that matter. This is also why I always consider the worst outcome of any decision, and only if I am ok with it is when I take the risk.</p>
