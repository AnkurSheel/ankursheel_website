    <h1 class="text-center">How to deploy a Gatsby site on a schedule with Azure Pipelines</h1>

<html><head></head><body><p>In an <a href="deploy-gatsby-site-netlify-azure-pipelines">earlier post</a>, I showed how to use azure pipelines to build a gatsby site and deploy to Netlify. But it was slightly limited. A live build was deployed only when a push was made to <em>master</em>. There was no way to have scheduled posts go live automatically.</p>
<p>There are 2 ways to approach this problem.</p>
<ol>
<li>By using an AWS Lambda or Azure Functions. I initially had this, and you can see the code <a href="https://github.com/AnkurSheel/BlogFunctions">here</a>.</li>
<li>By adding a new azure pipeline that runs on a schedule.</li>
</ol>
<p>In this post, I will be using azure pipelines to deploy the site daily.</p>
<p><em>Note: We will be using the template files created in the <a href="deploy-gatsby-site-netlify-azure-pipelines">Deploying a Gatsby site to Netlify using Azure Pipelines</a>, so I would suggest reading that first. Go on, I will wait.</em></p>
<h2 id="adding-a-new-pipeline-to-schedule-posts">Adding a new pipeline to schedule posts</h2>
<p>Good, you are back.</p>
<p>Let's create a new file <em>azure-pipelines.daily.yml</em>.</p>
<pre><code class="language-yaml hljs"><span class="hljs-attr">name:</span> <span class="hljs-string">'Daily Release'</span>

<span class="hljs-attr">schedules:</span>
<span class="hljs-attr">    - cron:</span> <span class="hljs-string">'0 14 * * *'</span>
<span class="hljs-attr">      displayName:</span> Daily build
<span class="hljs-attr">      branches:</span>
<span class="hljs-attr">          include:</span>
<span class="hljs-bullet">              -</span> master
<span class="hljs-attr">      always:</span> <span class="hljs-literal">true</span>

<span class="hljs-attr">trigger:</span>
<span class="hljs-attr">    batch:</span> <span class="hljs-literal">false</span>
<span class="hljs-attr">    branches:</span>
<span class="hljs-attr">        exclude:</span>
<span class="hljs-bullet">            -</span> <span class="hljs-string">'*'</span>

<span class="hljs-attr">pr:</span> none

<span class="hljs-attr">variables:</span>
<span class="hljs-attr">    - template:</span> azure-pipelines.vars.yml

<span class="hljs-attr">jobs:</span>
<span class="hljs-attr">    - template:</span> azure-pipelines.common.yml
<span class="hljs-attr">      parameters:</span>
<span class="hljs-attr">          deployScriptParams:</span> <span class="hljs-string">'--prod'</span>
<span class="hljs-attr">          netlifySiteId:</span> $(netlify.siteId)
<span class="hljs-attr">          netlifyToken:</span> $(netlify.token)
<span class="hljs-attr">          message:</span> <span class="hljs-string">'Daily build'</span>
</code></pre>
<ol>
<li><strong><em>name</em></strong>: The name for this pipeline.</li>
<li><strong><em>schedules</em></strong>: The schedule on which the pipeline should run.
<ol>
<li><strong><em>cron</em></strong>: The cron syntax defining the schedule.</li>
<li><strong><em>displayName</em></strong>: The display name for the schedule.</li>
<li><strong><em>branches</em></strong>: Include only the <em>master</em> branch so that only the <em>master</em> branch is built and deployed.</li>
<li><strong><em>always: true</em></strong>: Always run the pipeline.</li>
</ol>
</li>
<li><strong><em>trigger</em></strong>: Conditions for running the pipeline when a push is made.
<ol>
<li><strong><em>branches</em></strong>: Exclude all branches so that the build is not triggered for commits on any branch, including <em>master</em>.</li>
</ol>
</li>
<li><strong><em>pr: none</em></strong>: Disable triggering the pipeline when a PR is opened or changes are pushed to an open PR. <em>master</em>.</li>
<li><strong><em>variables</em></strong>: We set a template file to get the environment variables.</li>
<li><strong><em>jobs</em></strong>: The jobs to run.
<ol>
<li><strong><em>template</em></strong>: Use the job template created in <em>azure-pipelines.common.yml</em>.</li>
</ol>
</li>
<li><strong><em>parameters</em></strong>: The parameters to be passed to the job template
<ol>
<li><strong><em>netlifySiteId</em></strong>: The site ID from environment variables.</li>
<li><strong><em>netlifyToken</em></strong>: The token from environment variables.</li>
<li><strong><em>message</em></strong>: The message is hardcoded to Daily Build.</li>
</ol>
</li>
</ol>
<h2 id="gotchas">Gotchas</h2>
<ol>
<li>The cron syntax is mm HH DD MM DW. You can find more details about the supported syntax <a href="https://docs.microsoft.com/en-us/azure/devops/pipelines/build/triggers?view=azure-devops&amp;tabs=yaml#supported-cron-syntax">here</a></li>
<li>The timezone for schedules is UTC, so the build times need to be adjusted accordingly.</li>
<li>Don't forget to set <code>always: true</code>. If you have it set to false, a build might not be triggered if you schedule posts a few days in advance.</li>
</ol>
<h2 id="conclusion">Conclusion</h2>
<p>With this setup, I can write posts and schedule them to go live in the future.</p>
