<html><head></head><body><h2 id="problem">Problem</h2>
<p>I have two accounts from which I access my GitHub repositories. I have set one of them as a global setting. On a per-repository basis, I sometimes override the default details with my other account. Sometimes, I forget to override the default values and realize it only after making a few commits.</p>
<p>This post will show how we can update the author after making a few commits with incorrect details.</p>
<h2 id="solution">Solution</h2>
<p>First, we need to update our gitconfig with the author details.</p>
<pre><code class="language-txt">[user]
name = author_name
email = author_email
</code></pre>
<p>We can run the following command.</p>
<pre><code class="language-bash hljs">git rebase -i &lt;commit_hash&gt; -x <span class="hljs-string">"git commit --amend --reset-author -CHEAD"</span>
</code></pre>
<ul>
<li><strong>git rebase -i</strong>: Runs git rebase in interactive mode, allowing altering individual commits in the process.</li>
<li><strong>&lt;commit_hash&gt;</strong>: The hash of the commit after which we want to update the author.</li>
<li><strong>-x</strong>: Append the shell command for each line creating a commit.</li>
<li><strong>git commit --amend</strong>: Modify the most recent commit.</li>
<li><strong>--reset-author</strong>: Resets the author to the settings in the .gitconfig.</li>
<li><strong>-CHEAD</strong>: -C takes the existing commit object and reuses the log message without allowing the user to edit it. HEAD refers to the current commit we are viewing. -CHEAD takes the message from the current commit without opening the editor.</li>
</ul>
<p>We will then be presented with an editor to confirm all the commits we want.</p>
<p>We can run the following command to update the author for all commits, including the root.</p>
<pre><code class="language-bash hljs">git rebase -i --root -x <span class="hljs-string">"git commit --amend --reset-author -CHEAD"</span>
</code></pre>
<h2 id="conclusion">Conclusion</h2>
<p>This will update the author details for all the specified commits. This is also why it's important to sign your commits so that no one else can attribute bad malicious commits to you. You can see how to do this by following the steps in the <a href="securing-git-commits-windows">guide to securing git commits from tricking you on Windows</a>.</p>
