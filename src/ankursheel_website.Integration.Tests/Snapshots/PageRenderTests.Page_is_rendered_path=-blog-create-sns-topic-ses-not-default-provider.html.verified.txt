<html><head></head><body><p>Let us assume that most of our infrastructure is in <em>ap-southeast-2 (Sydney)</em>. But, our Amazon Simple Email Service(SES) is set up in <em>us-west-2 (Oregon)</em>. We also want to create Amazon Simple Notification Service(SNS) topics, and Amazon Simple Queue Service (SQS) queues to handle bounce notifications.</p>
<p>Since SES is in a different region, our SNS topics also need to be in that region. So how do we tell terraform to create the topics in the SES region?</p>
<h2 id="solution">Solution</h2>
<h3 id="step-1">Step 1</h3>
<p>Create providers.</p>
<pre><code class="language-hcl">provider "aws" {
  region = "ap-southeast-2"
}

provider "aws" {
  region = "us-west-2"
  alias  = "ses_aws"
}
</code></pre>
<ul>
<li><strong>ap-southeast-2</strong>: The default provider configuration.</li>
<li><strong>us-west-2</strong>: This is the additional provider configuration. Setting the <strong><em>Alias</em></strong> allows us to create more than 1 provider.</li>
</ul>
<h2 id="step-2">Step 2</h2>
<p>For the resources we want to use the <em>us-west-2</em> region, we specify the <em>provider</em> parameter and set it equal to the <em>alias</em>.</p>
<p>If a <em>provider</em> parameter is not specified, it will fall back to using the provider that doesn't have an alias.</p>
<pre><code class="language-hcl">resource "aws_sqs_queue" "example-queue" {
  name                      = "example_queue"
}

resource "aws_sns_topic" "example-topic" {
  name     = "example_topic"
  provider = aws.ses_aws
}
</code></pre>
<p>In the above snippet, <em>example-queue</em> will be created in <em>ap-southeast-2</em> but, <em>example-topic</em> will be created in <em>us-west-2</em>.</p>
<h2 id="conclusion">Conclusion</h2>
<p>We can change the region by creating additional providers and specifying the <em>provider</em> parameter in a resource.</p>
