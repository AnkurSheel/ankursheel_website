<html><head></head><body><p>If you are importing a javascript dependency into your typescript app, you might run into this error.</p>
<p><em><strong>"Could not find a declaration file for module 'third-party-library-name'. 'third-party-library-name.js' has an 'any' type. Try npm install @types/third-party-library-name if it exists or add a new declaration (.d.ts) file containing declare module 'third-party-library-name';"</strong></em></p>
<p>If you read the error message, you can see that it suggests checking the DefinitelyTyped repository.</p>
<p>You will find the repository and install it using <em>npm</em> or _yarn_if you are lucky.</p>
<p>But what if you are not so lucky?</p>
<h2 id="solution-1">Solution #1</h2>
<p>Now, you will only see this error if you have <code>noImplicitAny: true</code> in your <em>tsconfig.json</em>. So you could get around this error is by setting <code>noImplicitAny: false</code>.</p>
<p>But, by doing this, you lose type-safety, so let's rule this out as a viable option.</p>
<h2 id="solution-2">Solution #2</h2>
<p>Add a local declaration file.</p>
<h3 id="steps">Steps</h3>
<p>So, how exactly do you do that?</p>
<ul>
<li>Create a <strong><em>types</em></strong> folder.</li>
<li>Edit tsconfig.json to use the typeRoots property so that the compiler can find the local declaration files. You will need it to include both your local folder as well as <em>node</em>_<em>modules/@types</em>. You will also want to check the local folder before the <em>node</em>_<em>modules</em> folder.</li>
</ul>
<pre><code class="language-json hljs">"compilerOptions": {
 ...
 "typeRoots": ["./types", "node_modules/@types"]
    }
</code></pre>
<ul>
<li>Add the property to exclude property in tsconfig so that it does not get compiled.</li>
</ul>
<pre><code class="language-json hljs">"exclude": [..., "types"]
</code></pre>
<ul>
<li>Add a <strong><em>third-party-library-name</em></strong> folder in the <em>types</em> directory.</li>
<li>Add an <em>index.d.ts</em> in that folder.</li>
<li>Add your TypeDefinitions. But if, like me, you want to prototype things or do not want to write a full custom type, you can declare the module by adding this to the <em>index.d.ts</em> file.</li>
</ul>
<pre><code class="language-typescript hljs"><span class="hljs-keyword">declare</span> <span class="hljs-keyword">module</span> 'third-party-library-name';
</code></pre>
<p>The only caveat is that you don't get any IntelliSense.</p>
<h2 id="conclusion">Conclusion</h2>
<p>Hopefully, this post will help you. You won't have to resort to either removing Typescript from your project or losing type-safety by adding <code>noImplicitAny: false</code> in your tsconfig.json.</p>
