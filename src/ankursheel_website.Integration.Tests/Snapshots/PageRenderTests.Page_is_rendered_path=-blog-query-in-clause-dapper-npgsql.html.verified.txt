<html><head></head><body><h2 id="scenario">Scenario</h2>
<p>We want to get details of users that match a given list of ids.</p>
<h3 id="the-sql">The SQL</h3>
<pre><code class="language-sql hljs"><span class="hljs-keyword">Select</span> *
<span class="hljs-keyword">FROM</span> <span class="hljs-keyword">users</span>
<span class="hljs-keyword">WHERE</span> <span class="hljs-keyword">id</span> <span class="hljs-keyword">IN</span> (<span class="hljs-number">1</span>, <span class="hljs-number">2</span>, <span class="hljs-number">3</span>)
</code></pre>
<h3 id="implementation-in-c">Implementation in C#</h3>
<pre><code class="language-csharp hljs">void GetUsers(IReadOnlyCollection&lt;int&gt; userIds)
{
    const sql = @"Select * FROM users
                WHERE id IN @UserIds"

    var parameters = new Dictionary&lt;string, object&gt;
                    {
                        { "UserIds", userIds }
                    };

    using (var connection = new NpgsqlConnection(_connectionString))
    {
        var users  = connection.Query&lt;User&gt;(
                        sql,
                        parameters.ToDynamicParameters())
                    .ToList();
    }
}
</code></pre>
<p><em>Notes:</em></p>
<ul>
<li><em>We are using <a href="https://www.npgsql.org/">NPGSql</a> and <a href="https://github.com/DapperLib/Dapper">Dapper</a>.</em></li>
<li><em>ToDynamicParameters is an extension method to transform the dictionary into Dapper.DynamicParameters</em></li>
</ul>
<h2 id="problem-1">Problem #1</h2>
<p>On running this, we get an error.</p>
<pre><code class="language-csharp hljs">System.NotSupportedException: Npgsql <span class="hljs-number">3.</span>x removed support <span class="hljs-keyword">for</span> writing
    a parameter with an IEnumerable <span class="hljs-keyword">value</span>, use .ToList()/.ToArray()
    instead.
</code></pre>
<p>Luckily, the error message is telling us what we need to do. So let's make userIds an Array.</p>
<pre><code class="language-csharp hljs"><span class="hljs-function"><span class="hljs-keyword">void</span> <span class="hljs-title">GetUsers</span>(<span class="hljs-params">IReadOnlyCollection&lt;<span class="hljs-keyword">int</span>&gt; userIds</span>)
</span>{
    <span class="hljs-comment">//unchanged</span>

    <span class="hljs-keyword">var</span> parameters = <span class="hljs-keyword">new</span> Dictionary&lt;<span class="hljs-keyword">string</span>, <span class="hljs-keyword">object</span>&gt;
                    {
                        { <span class="hljs-string">"UserIds"</span>, userIds.ToArray() }
                    };

    <span class="hljs-comment">// unchanged</span>
}
</code></pre>
<h2 id="problem-2">Problem #2</h2>
<p>We now get the error.</p>
<pre><code class="language-csharp hljs">Npgsql.PostgresException : <span class="hljs-number">42601</span>: syntax error at or near <span class="hljs-string">"$2"</span>
</code></pre>
<p>Unfortunately, this is a more cryptic error.</p>
<p>There are 2 changes we need to make to the SQL to fix this error.</p>
<ul>
<li>parens are needed around the parameter</li>
<li><strong>IN</strong> needs to be replaced with <strong>= ANY</strong></li>
</ul>
<p>Our resulting SQL now becomes</p>
<pre><code class="language-csharp hljs">void GetUsers(IReadOnlyCollection&lt;int&gt; userIds)
{
    const sql = @"Select * FROM users
                WHERE id IN @UserIds"
   // unchanged
}
</code></pre>
<h2 id="final-solution">Final Solution</h2>
<pre><code class="language-csharp hljs">void GetUsers(IReadOnlyCollection&lt;int&gt; userIds)
{
    const sql = @"Select * FROM users
                WHERE id IN @UserIds"

    var parameters = new Dictionary&lt;string, object&gt;
                    {
                        { "UserIds", userIds.ToArray() }
                    };

    using (var connection = new NpgsqlConnection(_connectionString))
    {
        var users  = connection.Query&lt;User&gt;(
                        sql,
                        parameters.ToDynamicParameters())
                    .ToList();
    }
}
</code></pre>
<p>This allows us to get details of users that match a given list of ids.</p>
