<html><head></head><body><p>Sometimes you have a commit in a repository that you want to bring over to another repository. We can follow these steps to do just that.</p>
<h3 id="create-a-patch">Create a patch</h3>
<p>In the source repository, we can create a patch that describes the commit by running.</p>
<pre><code class="language-bash hljs">git format-patch -1 &lt;sha&gt;
</code></pre>
<ul>
<li><strong>-1</strong>: The number of commits to create the patch for</li>
<li><strong>sha</strong>: The SHA of the commit</li>
</ul>
<h3 id="apply-the-patch">Apply the patch</h3>
<p>In the target repository, we can apply the patch with</p>
<pre><code class="language-bash hljs">git apply PATH_TO_PATCH --reject
</code></pre>
<ul>
<li><strong>reject</strong>: Applies the parts of the patch that are applicable and leaves the rejected hunks in corresponding *.rej files. We have to apply these manually.</li>
</ul>
<h2 id="references">References</h2>
<ul>
<li><a href="https://git-scm.com/docs/git-format-patch">git-format-patch Documentation</a></li>

</ul>
