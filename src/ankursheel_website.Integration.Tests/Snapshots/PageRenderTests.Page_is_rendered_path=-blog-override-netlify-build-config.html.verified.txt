<html><head></head><body><h2 id="problem">Problem</h2>
<p>As part of the <a href="migrating-gatsby-statiq">migration from Gatsby to Statiq</a>, I wanted to keep the original site running with gatsby but deploy the new branch using statiq.</p>
<h2 id="solution">Solution</h2>
<p>First, we need to allow branch deploys for our branch.</p>
<p><img src="/assets/images/posts/override-netlify-build-config/cover.png" alt="Build Setting UI"></p>
<p>In the above screenshot, my branch for the migration is named <em>statiq</em>.</p>
<p>To override the build settings set in the Netlify UI, we can add the following snippet to <em>netlify.toml</em></p>
<pre><code class="language-toml hljs">[build]
  command = "dotnet run --project ./subDir/"
  publish = "./subDir/output"
</code></pre>
<p>So, what's happening here?</p>
<ul>
<li><strong>Line 1</strong>: Apply the build settings globally and override anything set in the UI.</li>
<li><strong>Line 2</strong>: Use the <code>dotnet run</code> as the build command and specify the project path.</li>
<li><strong>Line 3</strong>: The relative directory to the root directory containing the deploy-ready HTML files and assets.</li>
</ul>
<h2 id="conclusion">Conclusion</h2>
<p>The advantage of updating the <em>netlify.toml</em> file is that when we merge our branch back to master, it will automatically use Statiq to build the site.</p>

<ul>
